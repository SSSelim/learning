# rest

- representational state transfer

- originated in Dr. Roy Fielding's dissertation
  - Architectural Styles and the Design of Network-based Software Architectures

- rest principles derive from Roy's work on the http 1.0/1.1, he was one of the principal authors

- unlike soap, rest isn't a standart
- unlike soap, payload can be in any format

- uses http and uri
  - soap just uses http
